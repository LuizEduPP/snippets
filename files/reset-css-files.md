# Reset All CSS Files with Empty Selectors

This snippet finds every `.css` file in your current folder and completely replaces its content. The new content will only be an empty CSS rule (selector), using the exact name of the file.

> ⚠️ **CRITICAL WARNING:** This script forcefully overwrites the contents of the files. You will lose any code that was written in them. Always commit your work to Git or make a backup before running.

## 🛠️ How it works under the hood

To understand the core Bash command:

1. **`for f in *.css;`**: Loops through every file in the current directory ending in `.css`. The variable `$f` holds the full filename (e.g., `button.css`).
2. **`"${f%.css}"`**: This is Bash string manipulation. It takes the file name and strips the `.css` extension from the end. So `button.css` becomes just `button`.
3. **`echo "... {}"`**: Prints out the stripped filename followed by empty braces, creating a valid CSS selector like `button {}`.
4. **`> "$f"`**: The standard redirect operator. It takes the text generated by the `echo` command and completely overwrites (`>`) the original file with it.

---

## 🚀 Quick Usage

### Full Reset (Overwrite mode)

```bash
for f in *.css; do \
    echo "${f%.css} {}" > "$f"; \
done
```

**Before execution:**
```css
/* button.css */
button { color: red; margin: 10px; }
```

**After execution:**
```css
/* button.css */
button {}
```

## 💡 Safest Variant: Append Mode

If you don't want to destroy the existing code, you can just *append* a new empty block to the bottom of the files. We just change the overwrite operator (`>`) to the append operator (`>>`).

```bash
for f in *.css; do \
    echo -e "\n${f%.css} {}" >> "$f"; \
done
```
